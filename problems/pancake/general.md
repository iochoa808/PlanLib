## Description

A stack of n pancakes of distinct sizes must be sorted from largest-on-bottom to smallest-on-top using only prefix reversals. A single action — flip — is parameterised by a position p and reverses all pancakes from the top of the stack down to position p. No other rearrangement is permitted. The planning task is to find a sequence of flips that transforms a given initial permutation into the sorted configuration; the optimal variant additionally minimises the number of flips.

The domain has a clean combinatorial structure: the n! reachable states correspond to all permutations of n elements and the state space forms a Cayley graph on the symmetric group Sₙ generated by all prefix transpositions. This makes Pancake Sorting an attractive benchmark for admissible heuristics, pattern databases, and optimal planners, because the state space grows rapidly (n! states) yet the branching factor is only n−1.

## History

The problem was posed in 1975 by Jacob E. Goodman under the pseudonym "Harry Dweighter" in the Problems section of the American Mathematical Monthly [Goodman75]. It attracted immediate interest from combinatorialists. Bill Gates and Christos Papadimitriou analysed the harder "burnt pancake" variant — each pancake has a burnt side that must end face-down — in 1979 [GatesPapadimitriou79], obtaining bounds that remain among Gates's most-cited research results.

The planning-domain formulation became a standard benchmark for admissible heuristics because the optimal solution length varies widely across instances of the same size. A landmark result by Bulteau, Fertin, and Rusu (2012) proved that computing the pancake number (minimum flips to sort a given permutation) is NP-hard [BulteauFertin12], providing theoretical justification for the difficulty of optimal planning in this domain.

## Variants

- [Typed Variant](#domain-typed-variant)
- [Extended Data Structures](#domain-extended-data-structures)
- Burnt Pancake (each pancake has a burnt face that must end face-down)
- Weighted Pancake (minimise total flip cost rather than number of flips)

The typed variant encodes the geometry through auxiliary `lte` and `flipat` predicates and uses PDDL 2.1 conditional effects and universal quantification. The extended-data-structures variant uses a non-standard language (PDDL-XDS) with array types and in-place array mutation to give a more compact domain description.
